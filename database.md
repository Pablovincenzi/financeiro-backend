# Modelagem inicial do banco de dados

## Objetivo

Estruturar a base do sistema financeiro com autenticacao, modulos operacionais, categorias de despesas, tags e o fluxo inicial de cartoes, PIX e valores a receber.

## Autenticacao

### pessoas

- `id` INT4 PK GENERATED BY DEFAULT AS IDENTITY
- `nome_completo` VARCHAR(150) NOT NULL
- `cpf` VARCHAR(14) NULL UNIQUE
- `data_nascimento` DATE NULL
- `telefone` VARCHAR(20) NULL
- `email` VARCHAR(150) NULL
- `ativo` BOOLEAN NOT NULL DEFAULT true
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

### usuarios

- `id` INT4 PK GENERATED BY DEFAULT AS IDENTITY
- `pessoa_id` INT4 NOT NULL UNIQUE FK -> `pessoas.id`
- `login` VARCHAR(80) NOT NULL UNIQUE
- `email_login` VARCHAR(150) NOT NULL UNIQUE
- `senha_hash` VARCHAR(255) NOT NULL
- `nivel_acesso` VARCHAR(20) NOT NULL DEFAULT 'usuario'
- `ultimo_login_em` TIMESTAMP NULL
- `ativo` BOOLEAN NOT NULL DEFAULT true
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

## Tags e categorias

### tb_tag

- `id` INT4 PK GENERATED BY DEFAULT AS IDENTITY
- `nome` VARCHAR(80) NOT NULL UNIQUE
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

### categorias_despesa

- `id` INT4 PK GENERATED BY DEFAULT AS IDENTITY
- `nome` VARCHAR(120) NOT NULL UNIQUE
- `data_inicio` DATE NOT NULL
- `data_fim` DATE NULL
- `observacoes` TEXT NULL
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

### categorias_despesa_usuarios

- `id` INT4 PK GENERATED BY DEFAULT AS IDENTITY
- `categoria_id` INT4 NOT NULL FK -> `categorias_despesa.id`
- `usuario_id` INT4 NOT NULL FK -> `usuarios.id`
- `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- `UNIQUE (categoria_id, usuario_id)`

## Modulos financeiros

### despesas

- `usuario_id` INT4 FK -> `usuarios.id`
- `categoria_id` INT4 NOT NULL FK -> `categorias_despesa.id`
- `tag_id` INT4 NULL FK -> `tb_tag.id`
- `cartao_id` INT4 NULL FK -> `cartoes.id`
- `descricao` VARCHAR(150) NOT NULL
- `valor` NUMERIC(14,2) NOT NULL
- `data_vencimento` DATE NOT NULL
- `data_pagamento` DATE NULL
- `forma_pagamento` VARCHAR(20) NOT NULL
- `meio_pagamento` VARCHAR(20) NULL
- `observacoes` TEXT NULL
- `status` VARCHAR(20) DEFAULT 'pendente'

Regras:

- `forma_pagamento = 'a_vista'` exige `meio_pagamento` com `dinheiro` ou `pix`
- `forma_pagamento = 'a_prazo'` exige `cartao_id`

## Observacoes de negocio

- cada registro pertence ao usuario autenticado
- toda despesa deve possuir uma categoria cadastrada
- receitas e despesas podem possuir uma tag para classificacao macro`r`n- receitas e despesas podem ser geradas em recorrencia mensal a partir de uma quantidade de parcelas, criando lancamentos futuros independentes
- despesas diferenciam pagamento `a_vista` e `a_prazo`
- pagamento `a_vista` aceita apenas `dinheiro` e `pix`
- pagamento `a_prazo` deve apontar para um cartao ativo do usuario
- categorias de despesa podem ser compartilhadas com mais de um usuario
