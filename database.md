# Modelagem inicial do banco de dados

## Objetivo

Estruturar a base do sistema financeiro com autenticacao, modulos operacionais, categorias de despesas, tags e o fluxo inicial de cartoes, PIX e valores a receber.

## Observacoes de negocio

- cada registro pertence ao usuario autenticado
- toda despesa deve possuir uma categoria cadastrada
- receitas e despesas podem possuir uma tag para classificacao macro
- categorias de despesa podem ser compartilhadas com mais de um usuario
- compras no cartao podem existir sem fatura vinculada no primeiro momento
- PIX separa entradas e saidas de transferencias instantaneas
- recebiveis ajudam a acompanhar valores previstos antes do recebimento real
- cadastro publico cria pessoa e usuario vinculados usando o email como login inicial
- parcelamentos ficam para a proxima etapa de desenvolvimento
